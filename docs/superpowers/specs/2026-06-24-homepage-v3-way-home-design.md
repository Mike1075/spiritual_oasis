# 心灵家园首页 v3 ·「一条回家的路」+ AI 回家向导 · 设计文档

- 日期：2026-06-24
- 背景：v1/v2 已上线。v2 是"痛点→觉醒"漏斗。Mike 给出更高维的真正主线：**首页要给每个人一条"心灵回家的路"**。本文是 v3 的源头真相。

## 0. 一句话定位
心灵家园 = **每个人的家**。不论你此刻迷失在哪里，这里都有一条回家的路；我们陪你找到回家的下一步。

## 1. 组织原则（Mike 原话提炼）
- **以"路径指引"为主**：首页主线是一条回家的路（四段）。
- **课程为辅助**：按渴望分组的支线课程。
- **AI 咨询为兜底**：不知道从哪开始 → 和 AI 助手聊，给一条回家路径指引。
- **接住所有人**：路本身接住所有人（无论卡在哪都有下一步）；AI 向导兜底接住迷路的人——**不评判、不贴职业标签、不开处方**，只温柔指一条回家的路（借鉴心镜"镜子"语气 + 路径指引）。
- **不做职业心灵卡盘**。职业版 compass 退回服务「面向未来教育 / MAS-Life」线（即模拟报告里它本该接的人）。

## 5. AI 回家向导（兜底接住 · 本次核心新件）
- **角色**：一位温柔的"回家向导"。不是顾问、不评判、不开处方；借鉴心镜"镜子"语气（先听见、不急着给答案），但比心镜多一件事——**给一条回家的路径指引**（此刻你最该走哪一段、配哪个陪伴/课程）。
- **接住所有人**：用户说此刻状态（迷茫/焦虑/关系/想探索真相/想解决钱…）→ 向导先承接，再温柔指路（映射到四段路 + 陪伴 + 支线）。
- **安全红线**（学心镜）：匿名、非治疗/医疗声明、自伤念头先转真人 + 心理援助热线。
- **技术**：复用 compass 的 `streamChat`（src/lib/minimax.ts，MiniMax M 系列，maxDuration 300）；新 prompt 模块 + 新 API route + 对话式 UI（可基于现有 AssistantWidget）。

## 6. 首页 v3 结构
1. **Hero · 回家的邀请**：暖光晨景；主标走"回家/每个人的家/不论迷失在哪都有路"的表述（痛点作引子不作主轴）；主 CTA「找到我回家的下一步」→ AI 向导；次 CTA「先看看这条路」。

5. **AI 回家向导入口**（兜底接住）。
6. **收尾**：再发一次回家的邀请。
